Experiences

District Technical Support Specialist

May 2017 - Present
North Sanpete School District

My primary purpose in this position is systems administration but also includes a wide variety of tasks involving general troubleshooting, education of staff on technical issues, network maintenance and so on. A large portion of my time is spent searching for potential cybersecurity threats, exploiting them to test for vulnerability, fixing the issue, and then responsibly disclosing the issue that was found.

Among the issues found, one of them was a significant vulnerability in our e-mail implementation. With this vulnerability, an attacker from anywhere in the world could spoof any e-mail address within the school district. The potential damage that this could cause would have been massive. The fix I implemented was to have an SPF record created to make spoofed e-mails go to the spam folder. Another fix was to implement encrypted e-mail for those in the district responsible to handle sensitive student and staff information.

Server Administrator

2016 - Present
Saturdays' Gravy

As a part-time server administrator for the web development company Saturdays' Gravy, I am solely responsible for standing up and maintaining web servers. In order to make my job simpler I implemented the dev/ops tool Ansible; enabling me to orchestrate changes among multiple servers simultaneously. Being the sole administrator of these servers that are exposed on the web, I constantly am updating these servers and monitoring them for vulnerabilities.

These servers are scanned and attacked almost over 1,000 times every day. In order to mitigate these attacks, I implemented the Intrusion Detection System OSSec. By running this system on each of the servers I receive alerts on a regular basis if there are any attacks of note. However, by running this system on the servers, even doing an aggressive network scan would alert the server to block traffic from the scanning IP address.

Web Advisor

2015 - 2016
Bluehost

As a web advisor, my responsibility was to provide support for websites that were hosted on Bluehost's platform. The servers running the websites were CentOS 6 (Linux) servers. The CMS that we primarily gave support for was Wordpress, as such I had to become very familiar with Wordpress' relationship to the server and as a web application. Through this job I learned how to access web servers via SSH and became proficient in the BASH based command line on Linux and OS X. I also had to become very familiar with networking protocols including SSH, HTTP, HTTPS and SMTP. I also learned how DNS worked, how to troubleshoot it and how to manage it.

Dealing with Wordpress sites also required me to learn the basics about HTML, CSS and PHP. I became very familiar with basic tasks on the MySQL command line in order to more easily interact with the database powering Wordpress.
